    关于建一个自己的博客网站本来不打算记录的，因为感觉网上的资料一大堆，又不是很难，但是后面发现坑还挺多的，所以就记录下吧，万一自己后面还想弄的话，有资料查询

    # 总结本文的重点: 项目本地搭建 —— 创建github仓库 --- 获取clientId clientSecret --- 上传代码到仓库并部署到github上

    # 本地初步搭建

    mkdir blogs  //创建一个blogs文件夹  有git 文件夹则不需要执行此命令
cd blogs    //进入文件夹 有git 文件夹则不需要执行此命令
npm init -y   //npm 初始化
npm install -D vuepress //安装vuepress依赖
npm install @vssue/vuepress-plugin-vssue -D //添加评论插件
npm install  @vssue/api-github-v3 -D //添加github-v3版本 免登陆都可以评论

    需要注意的是我们编辑器的编码格式要改为UTF-8 防止乱码 完成以上这些我们就可以开始创建我们的文档了，先创建docs文件夹，然后在doc 文件夹下创建 README.md 文件，这个文件将存放我们的主页内容

    接下来 我们配置package.json 中scripts添加如下代码

    "scripts": {
  "docs:dev": "vuepress dev docs",
  "docs:build": "vuepress build docs"
}

    完成这些之后一个初步的博客框架就出来了，通过如下代码在本地启动服务器可以查看。

    npm run docs:dev

    当看到http://localhost:8080/的时候代表我们启动成功了

    # 创建github 仓库

    点击 new创建一个远端仓库 填写基本信息 获取ssh 地址信息

    # 文件目录

    在 docs 文件夹下添加 .vuepress 目录，所有 VuePress 相关的文件都会被放在这里。在 .vuepress 文件夹下添加 config.js，配置网站的标题和描述，handbook 里面配置子页面内容 此时项目的目录结构是这样的：

    docs/

    # 部署

    github 上建一个新仓库 注意修改脚本文件对应的用户名和仓库名： my-blog 目录下建立一个脚本文件：deploy.sh，脚本信息如下所示:

    #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io/<REPO>
git push -f git@github.com:LiangHaley/interview.git master:gh-pages

cd -

    执行 sh deploy.sh, 我们看到代码会被提交到远程仓库, 我们可以在仓库的 Settings -> Pages 中看到最后的地址： 最终页面显示如下:
